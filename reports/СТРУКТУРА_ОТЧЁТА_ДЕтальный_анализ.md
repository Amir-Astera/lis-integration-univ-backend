### Структура каждого блока:

**Part1 (строки N):**
- Строка N:   Заголовок `Рабочее место | Услуга | Амбулатория | ... | Стационар`
- Строка N+1: Названия отделений (Амбулатория + первые 8 Стационара)
- Строка N+2: Пустая (разделитель)
- Строки N+3...: Данные

**Part2 (строки M):**
- Строка M:   Заголовок `Рабочее место | Услуга | Стационар | ... | Всего`
- Строка M+1: Названия отделений (оставшиеся 15 Стационара)
- Строки M+2...: Данные

## 6. ПРАВИЛА ЧТЕНИЯ ДАННЫХ

### 6.1 Рабочее место (колонка B)
- Значение присутствует только в **первой строке** группы услуг
- Все последующие строки имеют `None` в колонке B
- **Требуется переменная carry-forward**: `current_workplace`

### 6.2 Услуга (колонка C)
- Заполнена в **каждой строке** с данными
- Пустая строка = разделитель между группами → пропускать

### 6.3 Числовые значения
- Пустая ячейка (`None`) = **0** (НЕ "нет данных"!)
- Числа могут быть `int` или `float` (1.0 вместо 1)
- Редко: строки "1||", "|||" → очищать через `.strip('|')`

### 6.4 Объединённые ячейки
- Excel использует merged cells для названий отделений
- openpyxl возвращает значение только в **top-left** ячейке
- Остальные ячейки = None (это нормально)

## 7. КРИТИЧЕСКИЙ АЛГОРИТМ СОПОСТАВЛЕНИЯ

### ⚠️ ГЛАВНОЕ ПРАВИЛО: Соединять Part1 и Part2 по КЛЮЧУ (workplace, service), НЕ по позиции строки!

**Почему:** Когда рабочее место разбито на несколько пар блоков (Клиническая химия — 4 пары), последняя услуга может иметь:
- данные Амбулатории в Part1 пары N+1
- данные Стационара в Part2 пары N

Это приводит к смещению на 1 строку.

**Правильный алгоритм:**

```python
# Шаг 1: Парсим ВСЕ Part1-блоки → собираем ambu данные
ambu_data = {}  # ключ: (workplace, service) → {dept_name: quantity}

# Шаг 2: Парсим ВСЕ Part2-блоки → собираем stac данные  
stac_data = {}  # ключ: (workplace, service) → {dept_name: quantity, stac_total, grand_total}

# Шаг 3: Объединяем по ключу (workplace, service)
for key in set(ambu_data) | set(stac_data):
    record = {**ambu_data.get(key, {}), **stac_data.get(key, {})}
```

## 9. ОСОБЕННОСТИ И ЛОВУШКИ

| Проблема | Причина | Решение |
|----------|---------|---------|
| None в колонке B | Продолжение рабочего места | carry-forward переменная |
| None в данных | Нет исследований = 0 | `val if val else 0` |
| "1\|\|" в ячейке | Артефакт экспорта | `.strip('\|')` |
| пмсп vs ПМСП | Два разных отделения | Хранить как есть |
| "Не определено" | Валидное отделение | Не фильтровать! |
| Строка Part2 не совпадает с Part1 | Рабочее место разбито на несколько пар | JOIN по (workplace, service) |
| 13 амбулаторных + 8 стационарных в Part1 | Физическое ограничение ширины | Объединять по ключу |
