## 2. КЛЮЧЕВАЯ КОНЦЕПЦИЯ: СИСТЕМА «ПАРА БЛОКОВ»

Логически каждая запись — это строка вида:
`(Рабочее место, Услуга, отделение1: кол-во, отделение2: кол-во, ...)`

Но все 29 отделений не помещаются в ширину одного экрана, поэтому каждая такая строка **физически разрезана на два полублока**:

- **Блок ЧАСТЬ-1 (Part1)** — содержит Амбулаторию (13 отд.) + первые 8 отделений Стационара
- **Блок ЧАСТЬ-2 (Part2)** — содержит оставшиеся 15 отделений Стационара + итоги

**Part1 и Part2 всегда идут парой.** В файле таких пар 10 штук (по 2 строки-заголовка каждая), они покрывают все 7 рабочих мест.


## 5. СТРОКИ ДАННЫХ: ПРАВИЛА ЧТЕНИЯ

### 5.1 Рабочее место (col B)
- Значение присваивается только первой строке группы услуг
- Последующие строки той же группы имеют `None` в col B
- Нужна переменная `current_workplace` с переносом вниз
- Рабочее место может включать несколько строк с одним значением через `merged cell` (B8:B23 = 'гематология' означает, что все строки 8–23 относятся к гематологии)

### 5.2 Услуга (col C)
- Всегда заполнена у строк с данными
- Пустая строка (все колонки None) = разделитель между группами → **пропускать**

### 5.3 Числовые значения
- Пустая ячейка (`None`) = **0** (не значит «нет данных»!)
- Числа могут быть `int` или `float` (1.0 вместо 1) — конвертировать в `int`
- Редко встречаются строки «1||», «|||» — артефакты экспорта: `.strip('|')` и парсить

### 5.4 Объединённые ячейки в строках данных
- Иногда значение записано в merged cell (например AH:AI объединены для одного числа)
- openpyxl возвращает значение в top-left ячейке, остальные = None
- Это **не проблема**: просто игнорируй None-ячейки, читай только ненулевые значения по карте столбцов

## 6. КРИТИЧЕСКИЙ АЛГОРИТМ ПАРСИНГА

### ⚠️ ГЛАВНОЕ ПРАВИЛО: Соединять Part1 и Part2 по КЛЮЧУ, НЕ по позиции строки!

**Почему нельзя по позиции:** Когда рабочее место занимает много услуг и переходит через несколько пар блоков (например Клин.химия занимает 4 пары, Общеклинические — 2 пары), последняя услуга может иметь:
- данные Амбулатории в Part1 пары N+1
- данные Стационара в Part2 пары N

Это приводит к смещению на 1 строку и неверному сопоставлению.

**Правильный подход:**
```python
# Шаг 1: Парсим ВСЕ Part1-блоки → собираем ambu данные
ambu_data = {}  # ключ: (workplace, service) → {dept_name: quantity}

# Шаг 2: Парсим ВСЕ Part2-блоки → собираем stac данные  
stac_data = {}  # ключ: (workplace, service) → {dept_name: quantity, stac_total, grand_total}

# Шаг 3: Объединяем по ключу (workplace, service)
for key in set(ambu_data) | set(stac_data):
    record = {**ambu_data.get(key, {}), **stac_data.get(key, {})}
```

### Алгоритм разбора одного блока

```python
def parse_block(ws, header_row, block_type):
    """
    header_row: строка где col B='Рабочее место' (1-indexed)
    block_type: 'part1' или 'part2'
    """
    # Строка с именами отделений — следующая после заголовка
    dept_row_idx = header_row + 1
    
    # Читаем карту столбцов из строки отделений
    col_to_dept = {}
    for col in range(1, 40):
        val = ws.cell(dept_row_idx, col).value
        if val is not None and str(val).strip():
            dept_name = str(val).strip()
            # Пропускаем служебные столбцы
            if dept_name not in ('Рабочее место', 'Услуга', 'Амбулатория', 
                                  'Стационар', 'Всего'):
                col_to_dept[col] = dept_name
    
    # Определяем тип каждого отделения и исключаем итоговые столбцы
    if block_type == 'part1':
        total_cols = {26, 27}      # Всего Амбулатория (Z:AA)
        ambu_cols = set(range(4, 28))   # D-AA
        stac_cols = set(range(28, 39))  # AB-AL
    else:  # part2
        total_cols = {32, 33}     # Всего Стационар (AF), Всего общее (AG)
        stac_cols = set(range(4, 32))   # D-AE
        ambu_cols = set()
    
    # Читаем данные
    current_workplace = None
    results = []
    
    # Данные начинаются через 3 строки после header_row (header + 2 строки заголовка отделений)
    # Но заголовок занимает merged cells через строки N, N+1, N+2 → данные с N+3
    # Определяем конец блока: следующий header_row или конец листа
    
    for data_row in range(header_row + 3, ws.max_row + 1):
        row_vals = {c: ws.cell(data_row, c).value for c in range(1, 40)}
        
        # Признак конца блока: наткнулись на следующий заголовок
        if row_vals.get(2) == 'Рабочее место':
            break
        
        # Пустая строка — разделитель
        service = row_vals.get(3)
        if service is None:
            continue
        
        # Обновляем рабочее место
        if row_vals.get(2) is not None:
            current_workplace = str(row_vals[2]).strip()
        
        # Собираем значения по отделениям
        row_data = {
            'workplace': current_workplace,
            'service': str(service).strip(),
        }
        
        for col, dept_name in col_to_dept.items():
            if col in total_cols:
                continue  # пропускаем итоговые
            val = row_vals.get(col)
            quantity = parse_qty(val)
            dept_type = 'ambulatory' if col in ambu_cols else 'stationary'
            row_data[f'{dept_type}|{dept_name}'] = quantity
        
        # Сохраняем итоги из Part2
        if block_type == 'part2':
            row_data['total_stationary'] = parse_qty(row_vals.get(32))
            row_data['total_grand'] = parse_qty(row_vals.get(33))
        else:
            row_data['total_ambulatory'] = parse_qty(row_vals.get(26))
        
        results.append(row_data)
    
    return results


def parse_qty(val):
    if val is None:
        return 0
    s = str(val).strip().strip('|').strip()
    if not s:
        return 0
    try:
        return int(float(s))
    except:
        return 0
```

## 14. ПОДВОДНЫЕ КАМНИ

| Проблема | Причина | Решение |
|----------|---------|---------|
| Строка Part2 не совпадает с Part1 по индексу | Рабочее место разбито на несколько пар | JOIN по (workplace, service), не по позиции |
| None в col B | Продолжение предыдущего рабочего места | carry-forward переменная |
| Одно число в двух соседних столбцах выглядит как один | openpyxl возвращает значение только в top-left ячейке merge | Читать только top-left; остальные будут None |
| «Не определено» кажется ошибкой | Реальное отделение с таким названием | Не фильтровать! |
| пмсп vs ПМСП | Два разных отделения с похожими названиями | Хранить оба, не объединять |
| Числа как float (1.0) | Формат ячеек Excel | Приводить через `int(float(val))` |
